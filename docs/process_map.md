# Process Map — Manual Reconciliation (BEFORE) vs FinRecon Copilot (AFTER)

This is the "analyze the process, spot the automation opportunity" deliverable
from Phase 0. Use it in the interview to show you understood the *business*
before you wrote a line of code. A rendered image is at `process_map.png`
(generated by `make_process_map.py`).

## BEFORE — manual reconciliation (slow, error-prone)

```
 Analyst downloads    Analyst downloads     Break notes arrive
   ledger export        bank statement       by email / PDF
        |                    |                     |
        v                    v                     v
 ┌────────────────────────────────────────────────────────┐
 │  Eyeball / VLOOKUP in Excel to line up references        │  ~hours
 └───────────────────────────┬────────────────────────────┘
                             v
 ┌────────────────────────────────────────────────────────┐
 │  Manually spot breaks; copy-paste; re-key amounts        │  error-prone
 └───────────────────────────┬────────────────────────────┘
                             v
 ┌────────────────────────────────────────────────────────┐
 │  Read each note by hand to guess WHY the break happened  │  buried context
 └───────────────────────────┬────────────────────────────┘
                             v
 ┌────────────────────────────────────────────────────────┐
 │  Email a manager for sign-off; no consistent audit trail │  weak controls
 └────────────────────────────────────────────────────────┘

 Pain: multi-hour task, missed breaks, inconsistent categorisation,
       no reproducible audit trail, "why" buried in messy notes.
```

## AFTER — FinRecon Copilot (minutes, controlled, audit-ready)

```
 ledger.csv          bank_feed.csv          remittance notes (txt/PDF)
     |                    |                          |
     └──────────┬─────────┘                          │
                v                                     │
   SQL + Python reconcile (exact, testable)  ->  BREAKS
                                                      │
                                                      v
                          LLM reads each note (temp 0): classify + extract + summarise
                                                      │
                                                      v
              Governance: mask PII -> validate -> audit log -> human approves the ones that matter
                                                      │
                                                      v
        Power Automate schedules + routes approval + alerts   ->   Power BI dashboard
```

## What changed (the talking points)

| Dimension        | Before                        | After                                  |
|------------------|-------------------------------|----------------------------------------|
| Time             | Multiple hours, manual        | Minutes, one scheduled run             |
| Matching         | Excel VLOOKUP, human error    | Deterministic SQL/Python, tested       |
| "Why" of a break | Read notes by hand            | LLM classifies + extracts, validated   |
| Controls         | Ad-hoc email sign-off         | PII masking, validation, audit log, HITL|
| Reporting        | Static spreadsheet            | Live Power BI dashboard                |
| Repeatability    | Different every analyst       | Same result every run (temp 0 + rules) |

> Have a **number** ready for the interview (even an estimate): e.g. "turned a
> ~3-hour daily task into a ~5-minute reviewed run", "classifier ~90% accurate",
> "X% of low-value breaks auto-handled, rest routed to a human".
